1 系统安装时无法选择分区

当系统安装进行到“where do you want to install windows"时，出现提示信息：“Setup was unable to create a new system partition or locate an existing system partition.See the setup log files for more information."

安装程序无法创建新的系统分区，也无法定位现有系统分区

(原因：在pe下格式化c盘之后，pe的内核一般是2003的内核，是不能安装win7。)

解决办法：模拟光盘安装的关键的系统文件和环境：

1.1 下载win7系统镜像并解压缩到你想安装的系统盘（如C：盘）外的其它盘上（如E：）。

1.2 重启机器，通过U盘启动（做一个可U盘启动的PE系统，如老毛桃），进入win pe系统。

1.3 格式化C:盘(NTFS文件系统）；

1.4 在解压的安装程序的文件夹中找到文件夹boot和sources，文件bootmgr复制到C盘根目录下。

1.5 运行cmd ，输入 c:\boot\bootsect.exe /nt60 c: ，然后回车，看到提示successful字样的语句即成功了！然后重启电脑。

注：bootsect.exe（boot sector）是Windows vista/7中光盘中自带的一个程序，字面解释是引导扇区修复工具（boot sector restoration tool），为磁盘分区升级主引导编码，以转换bootmgr与NTLDR。用该工具可以修复在你电脑上的启动扇区。/nt60 涂写与BOOTMGR一致的主引导编码到(SYS，ALL，Driveletter)分区。 装在(SYS，ALL，DriveLetter)分区的操作系统必须是vista，2008，或以上版本。

1.6 把U盘从电脑中拔出，取出U盘，然后从硬盘启动，然后出现了win7 安装的欢迎界面，选择好地域、语言、键盘。点“下一步”，选择分区，这时，这个“安装程序无法定位现有系统分区，也无法创建新的系统分区”的提示没有出现，选择C 分区，安装OK 。

(主分区也称主磁盘分区，它是磁盘引导区所在的分区。它是包含操作系统启动所必需的的文件和数据的硬盘分区。简单的说只有主分区才能装系统。)

使系统更安全可靠的隐藏分区:

如果是裸机全新安装Windows 7，或者在安装时利用安装程序自带的分区工具重新进行了分区，安装完成后大家会在磁盘管理器中发现一个神秘的隐藏分区.这个隐藏分区的大小在Windows 7的各个版本中是不同的.

(1).当我们在Windows 7中启用了Bit-Locker或者在进行系统恢复时，这种分区模式为我们提供了便利，也保证了系统的安全。因为，Bit-Locker以及系统恢复都不会对该分区进行操作，从而保证在其中保存的系统启动引导文件免遭破坏，以保证计算机的正常引导。　　

(2).实现了系统引导文件和系统文件的隔离，避免的一方遭到破坏而殃及池鱼。通常，我们习惯上系统将整个Windows装在C盘中。其实，从技术上来说Windows的启动过程是分成两个步骤：Windows的System分区和Boot分区。其中System分区保存了Windows启动时所必须的启动文件，比如引导扇区(mbr)里的东西，相应的引导文件，第三方驱动比如对RID硬盘支持等等。所谓Boot分区，其实就是我们通常所说的C:中的东西，包含了Windows运行是所需要的绝大多数文件。所以说，windows的System分区是一个比较小的分区，而这个隐藏分区就是从通常的系统分区中隔离出去的Windows的System分区。　　　

(3).还有一个好处是显而易见的：当我们的系统出现故障，或者我们要做双系统的安装，以及启用Bit-Locker时，将引导相关的文件保存在该隐藏分区中就避免了在进行上述操作时对System分区的修改或者破坏，使得所有的操作都是针对Windows的Boot分区的.
